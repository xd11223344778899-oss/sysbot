# SysBot

بوت Discord نظامي متعدد السيرفرات، مبني بـ TypeScript و discord.js v14، بمعمارية وحدات قابلة للتوسع. كل أمر هو وحدة مستقلة تُسجَّل تلقائياً، والتفاعل يتم عبر الأوامر النصية والقوائم التفاعلية (بدون Slash).

راجع [PROJECT_SPEC.md](PROJECT_SPEC.md) للمواصفات الكاملة وقائمة الأوامر.

## المتطلبات

- Node.js 20+
- لا حاجة لأي خدمة خارجية: قاعدة البيانات محلية (SQLite، ملف واحد) ولا يوجد Redis.
- تطبيق بوت في Discord مع تفعيل الـ Privileged Intents: `Server Members`, `Message Content`, `Presence`.

## الإعداد المحلي (هذا الجهاز)

```bash
cp .env.example .env
# املأ DISCORD_TOKEN و GLOBAL_OWNERS فقط

npm install            # يولّد عميل Prisma تلقائياً (postinstall)
npm run dev            # ينشئ/يحدّث قاعدة البيانات ثم يشغّل البوت
```

قاعدة البيانات تُنشأ تلقائياً كملف في `data/sysbot.db`. لا تحتاج تثبيت أي قاعدة بيانات.

للتشغيل بدون مراقبة التغييرات:

```bash
npm run build
npm start
```

أدوات مساعدة: `npm run db:studio` لتصفّح البيانات، `npm run db:push` لمزامنة السكيما.

## النشر على Railway

1. اربط المستودع على GitHub بمشروع Railway (New Project → Deploy from GitHub → اختر `sysbot`).
2. Railway يبني عبر [Dockerfile](Dockerfile) (`npm run build`) ثم يشغّل `npm run start:prod` (ينشئ/يحدّث SQLite على Volume ثم يشغّل البوت).
3. أضف **Volume** على المسار `/data` (إلزامي — بدونه تُفقد الإعدادات عند كل deploy). لا تستخدم `VOLUME` في Dockerfile؛ Railway يوفّر Volumes من لوحة التحكم فقط.
4. اضبط متغيرات البيئة:

| المتغير | القيمة |
|---------|--------|
| `DISCORD_TOKEN` | توكن البوت من Developer Portal |
| `GLOBAL_OWNERS` | معرفات الأونرات (مفصولة بفاصلة) |
| `DATABASE_URL` | `file:/data/sysbot.db` |
| `LOG_PRETTY` | `false` |
| `LOG_LEVEL` | `info` |

5. بعد النشر، راقب Logs: يجب أن تظهر `Connected to local database` ثم `Bot is ready`.

لا حاجة لفتح HTTP port — البوت يعمل كـ worker طويل الأمد عبر Discord WebSocket.

سياسة `restartPolicyType: ALWAYS` في [railway.json](railway.json) تجعل أمر `restart` يعيد التشغيل فعلياً (العملية تخرج فيعيدها Railway).

> مهم: بدون Volume على `/data`، ملف SQLite يُمسح عند كل إعادة نشر.

## التشغيل عبر Docker (اختياري)

```bash
cp .env.example .env      # املأ DISCORD_TOKEN و GLOBAL_OWNERS
docker compose up -d --build
```

يشغّل البوت فقط، مع Volume باسم `botdata` على `/data` لحفظ قاعدة البيانات. `restart: unless-stopped` يدعم أمر إعادة التشغيل.

## أول تشغيل في سيرفر

1. أضف البوت بصلاحية Administrator (و**Mute Members** لتفعيل `vmute`).
2. اكتب `!vip` ثم اضغط "تشغيل الإعداد الأولي" — سيُنشئ الرولات وقنوات اللوق ويطبّق صلاحيات الإسكات ويزرع أسباب العقوبات الافتراضية.
3. غيّر البرفكس عند الحاجة عبر `!setprefix` أو فعّل وضع بدون برفكس عبر `!setnprefix`.

## أسماء الأوامر البديلة

كل أمر له اسم عربي مختصر افتراضي (مثل `اسكت` لـ `mute`). يمكن تخصيصها لكل سيرفر:

| الأمر | الوصف |
|--------|--------|
| `doadd <أمر> <اسم>` | إضافة alias إضافي |
| `dochange <أمر> <اسم>` | استبدال الاسم العربي/الرئيسي |
| `doremove <أمر> <اسم>` | حذف alias مخصص |
| `dolist [أمر]` | عرض الأسماء (افتراضي + مخصص) |

الاسم الإنجليزي الأساسي يبقى يعمل دائماً.

## قوائم أسباب العقوبات

عند كتابة `mute @user` (أو `prison` / `vmute` / `ban` / `kick`) **بدون مدة أو سبب**، يظهر Embed مع قائمة Select Menu بالأسباب الجاهزة ومدة كل سبب. خيار «سبب مخصص» يفتح Modal للإدخال اليدوي.

إدارة الأسباب: `resons add 30m سبام mute prison`، `resons list`، `resons edit`، `resons remove`.

## كتم صوتي (vmute)

`vmute` يستخدم **Server Mute** (`setMute`) فقط — بدون رول. طوال مدة العقوبة، إذا فك العضو الكتم يدوياً أو دخل روم صوتي، يعيد البوت الكتم تلقائياً. **لا يمنع الكتابة في الشات.**

## إسكات كتابي (mute)

`mute` يمنح العضو رول **Muted** الذي يمنع الكتابة في جميع القنوات النصية (العامة، الخاصة، والمدمجة مع الرومات الصوتية). **لا يؤثر على التحدث في الصوت.**

يُطبَّق رول Muted على صلاحيات كل القنوات عند الإعداد الأولي (`lsetup`). لإصلاح صلاحيات خاطئة (مثل Speak) دون تكرار اللوقات: `!lsetup sync`.

## البنية

```
src/
  core/        محرك الأوامر: parser, registry, permission engine
  database/    Prisma (SQLite) client + خدمة إعدادات السيرفر
  services/    setup, penalties, penalty-scheduler, role-bulk, logging, member gate, auto-moderation
  events/      مستمعو أحداث Discord (لوقات، حماية، بوابة الأعضاء)
  modules/     الأوامر مقسّمة حسب الفئة (كل ملف يصدّر مصفوفة أوامر)
  shared/      ثوابت، إمبدات، مُحلِّلات وسائط، أنواع
```

### إضافة أمر جديد

أنشئ كائن `Command` في الوحدة المناسبة وأضفه إلى مصفوفة التصدير — سيُسجَّل تلقائياً ويظهر في `commands`.

فيما يلي **جميع الأوامر المسجّلة في البوت** (138 أمراً)، مرتبة حسب التصنيف. البرفكس الافتراضي `!` ويمكن تغييره عبر `setprefix`.

**أسماء بديلة:** كل أمر له alias عربي افتراضي (مثل `اسكت` = `mute`). تخصيص عبر `doadd` / `dochange` / `doremove` / `dolist`. الأوامر القديمة: `help` = `ping`، `reset` = `preset`، `play` = `setactivity`

**أوامر aliases:** `doadd`، `dochange`، `doremove`، `dolist`
